# AI Daily Radar 功能说明

## 原始需求

这个项目的目标，是做一个轻量级的 AI 更新雷达系统。

核心诉求有 4 个：

1. 每天定时抓取指定网站的最新更新内容
2. 使用 LLM 对抓取结果做提炼、过滤和摘要
3. 去掉重复内容，只保留值得关注的更新
4. 最终通过企业微信机器人把结果发送出来

原始需求里的技术方向是：

- 运行环境使用 Node.js
- 定时任务使用 GitHub Actions
- 网页抓取使用 Jina Reader API
- 模型调用使用 OpenAI Compatible API
- 消息通知使用企业微信 Webhook

原始需求里的输出目标是：

- 每天生成一条 AI 雷达消息
- 每条消息包含产品名、功能简介、亮点说明
- 当没有有效更新时，发送“今日暂无重大更新”

## 技术实现

当前项目已经实现为一个可以直接运行和定时执行的 Node.js 工具。

### 1. 数据来源

目前默认抓取两个 changelog 页面：

- GitHub Copilot Changelog
- Cursor Changelog

也支持通过环境变量 `RADAR_TARGET_URLS` 自定义来源。

### 2. 抓取方式

项目会把目标页面交给 `https://r.jina.ai/{URL}` 转成 Markdown，再交给后续流程处理。

为了避免 changelog 页面一次抓出太多历史内容，当前实现对这类页面做了特别处理：

- 只提取每个来源的最新一条更新
- 尽量拿到该条更新的官方发布时间
- 优先使用更新详情页的直达链接，而不是栏目页链接

### 3. LLM 处理逻辑

模型会根据抓到的最新内容，生成结构化结果，字段包括：

- `name`: 更新名称
- `summary`: 20 字内的功能概括
- `highlight`: 一句话亮点
- `publishedAt`: 官方发布时间，如果页面里有的话

同时做了几层稳定性处理：

- 限制输入长度，避免内容过长
- 对免费模型的瞬时失败做自动重试
- 对 LLM 返回的坏 JSON 做容错修复

### 4. 去重逻辑

项目会把历史结果写入 `data/history.json`。

当前的去重规则是：

- `name + publishedAt + sourceUrl`

这样可以避免同名但不同发布日期的更新被错误去重。

### 5. 通知方式

最终消息会以企业微信 Markdown 的形式发送。

当前消息内容包括：

- 更新标题
- 发布时间（如果有）
- 功能简介
- 亮点说明
- 原文可点击链接

### 6. 自动化运行

项目已经集成 GitHub Actions：

- 每天 UTC 01:00 自动执行
- 对应北京时间每天 09:00
- 也支持手动触发

同时工作流里已经补充了：

- 依赖安装
- 代码检查
- `history.json` 的缓存持久化

### 7. 当前状态

本项目当前已经具备以下能力：

- 本地可运行
- 企业微信可推送
- changelog 只取最新一条
- 发布时间可提取并展示
- GitHub Actions 可自动定时执行
- 历史记录可用于去重
